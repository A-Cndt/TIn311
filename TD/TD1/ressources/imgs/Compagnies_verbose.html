Avion ( Immatriculation, Type, #Numero_unique )
  • Le champ Immatriculation constitue la clé primaire de la table. C'était déjà un identifiant de l'entité Avion.
  • Le champ Type était déjà un simple attribut de l'entité Avion.
  • Le champ Numero_unique est une clé étrangère. Il a migré par l'association de dépendance fonctionnelle Possede à partir de l'entité Compagnie en perdant son caractère identifiant.
Compagnie ( Numero_unique, Nom )
  • Le champ Numero_unique constitue la clé primaire de la table. C'était déjà un identifiant de l'entité Compagnie.
  • Le champ Nom était déjà un simple attribut de l'entité Compagnie.
Embarquer ( #N°Vol, #ID )
  • Le champ N°Vol fait partie de la clé primaire de la table. C'est une clé étrangère qui a migré directement à partir de l'entité Vol.
  • Le champ ID fait partie de la clé primaire de la table. C'est une clé étrangère qui a migré directement à partir de l'entité Passager.
Passager ( ID, Nom, Prenom )
  • Le champ ID constitue la clé primaire de la table. C'était déjà un identifiant de l'entité Passager.
  • Les champs Nom et Prenom étaient déjà de simples attributs de l'entité Passager.
Vol ( N°Vol, Ville_depart, Destination, Date_decollage, date_atterrissage, Nb_passagers, #Numero_unique )
  • Le champ N°Vol constitue la clé primaire de la table. C'était déjà un identifiant de l'entité Vol.
  • Les champs Ville_depart, Destination, Date_decollage, date_atterrissage et Nb_passagers étaient déjà de simples attributs de l'entité Vol.
  • Le champ Numero_unique est une clé étrangère. Il a migré par l'association de dépendance fonctionnelle Opere à partir de l'entité Compagnie en perdant son caractère identifiant.
